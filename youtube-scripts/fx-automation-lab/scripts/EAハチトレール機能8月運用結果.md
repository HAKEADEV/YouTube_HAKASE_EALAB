## １．ロジックの説明

ではまず、ロジックを知らない方の為におさらいです。

EAハチは、"ゴトー日"に買われやすいという円のアノマリーをもとに設計されたEAです。

・単ポジ型（常に1ポジずつ決済ロジックが入っている）
・月間トレード数は20回未満
・高頻度ではなく、精査されたエントリーが特徴

忠実に、一歩一歩積み上げるようなトレードを行います。

そして徹底した資金管理機能が入っており、自信で設定したロット以上は入らないようなリスクを抑えた設計です。

但し、単ポジ型であるがゆえ、月次毎にマイナスの月が出ることもあり、利益だったのに逃した...なんて事もあり

今回は前回まで検証していた、トレール。建値機能のフォワード結果の発表となります。

## ２．前回の振り返り

前回までの検証で、
建値やトレールの「開始タイミング」が結果を左右することがわかってきました。

特にEAハチでは、
**"トレールの発動を早めるか？遅らせるか？"**によって、
「伸びる利確を逃す or 逃さず取れる」のバランスが大きく変わってくるんです。

だからこそ──
「実際の運用でどうなったのか？」
「8月の相場では、どれが最適だったのか？」
ここをしっかりと見ていきたいと思います。

---

## ロジックの違いを整理（3つの比較軸）

今回比較するのは、EAハチの3つのバージョンです。

| バージョン      | 内容                       |
| ---------- | ------------------------ |
| オリジナル      | 通常の利確・損切ロジック。トレールなし。     |
| ハチ平均値トレール     | 含み益が一定pips出たら即トレール開始     |
| ハチトレール時間遅延型 | エントリーから時間遅延型後にトレール発動（時間遅延型） |

この違いによって「利をどこで確保するか？」が大きく変わってきます。

## 8月の運用結果の違い

実際に2025年8月の運用結果を比較すると…

| ロジック        | 損益       | 勝率     | 取引回数 |
| ----------- | -------- | ------ | ---- |
| オリジナル       | -29,938円 | 50.00% | 16回  |

| ハチトレール時間遅延型後 | +4,712円  | 50.00% | 16回  |

最も利益が出たのはハチトレール（即時型）。

## チャートから見る「差が出た日」

検証として、実際の相場で大きく結果が変わった取引日を3日ピックアップしました。

📅 8月20日
| ロジック        | 損益       |
| ----------- | -------- |
| オリジナル       | +16,804円 |
| ハチトレール      | +26,584円 |
| ハチトレール時間遅延型後 | +15,394円 |

3ポジション保有。

1ポジ目：全ロジック共通の利益。

2ポジ目：トレールで利を伸ばせた（特にハチトレール）。

3ポジ目：時間遅延型型だけトレール発動前の急騰で建値撤退。その後下落。

→ この日はトレールの「早さ」がそのままリターンに反映された典型例です。

📅 8月26日

| ロジック        | 損益       |
| ----------- | -------- |
| オリジナル       | +1,150円  |
| ハチトレール      | +25,100円 |
| ハチトレール時間遅延型後 | +25,100円 |

急落直後にトレールが発動して利確。

オリジナルはその後建値付近に戻ってきてしまい、利益を逃した。

→ トレールによる逃げ切りが成功した好例です。

📅 8月28～29日

| ロジック        | 損益      |
| ----------- | ------- |
| オリジナル       | +6,110円 |
| ハチトレール      | +9,210円 |
| ハチトレール時間遅延型後 | +3,110円 |

28日：時間遅延型後型はトレール発動前に建値撤退。

ハチトレールは早めの利確で利益確保。

29日：トレール組は利確成功、オリジナルは戻ってきて利を逃す。

→ 一連の流れから、**"トレールの早さ＝逃げの強さ"**という構図が見えてきました。

## ⑤ 考察：なぜ結果がこれだけ分かれたのか？

今回のように、

早めのトレール → 一時的な伸びをキャッチして逃げる

遅めのトレール → 確信が持てた時だけ発動

というように、"開始タイミング"だけで勝率も損益も大きく変わることが分かりました。

そして、今回のバックテストでは時間遅延型後トレール型が最も良い結果でしたが、
8月という限られた期間での結果であり、今後も安定するとは限りません。

だからこそ大事なのが──
「自分で選べる」「場面で使い分けられる」裁量の余地です。

---

## 考察と結論：「どう使い分けるべきか？」

今回の結果から見えてきたのは以下のポイントです。

✅ ハチトレール（即時型）が良かった理由：

早めの利確で"戻り"を避けられた

取引回数は同じでも勝率が上がったのは"含み益の確保率"が上がったから

ハチトレール時間遅延型型の特徴：

「じっくり待って伸ばす」設計

ただし急変時にトレールが間に合わないケースもあり

✅ オリジナルの特徴：

良くも悪くも粘る

だからこそ"利を伸ばすこともある"が、"逃すこともある"
