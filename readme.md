# はかせのFX自動売買研究所 - プロジェクト全体設計書

## 📋 プロジェクト概要

このリポジトリは、**「はかせのFX自動売買研究所」ブランド**を中核とする複合事業全体の設計・開発・運用を統括管理するためのプロジェクトです。

本プロジェクトでは、複数の事業レイヤーが相互に連携し、以下の3つの軸を常に意識しながら成長していきます：

- **収益性**：短期・中期・長期の収益構造の最適化
- **工数**：開発コスト・運用負荷のバランス調整
- **ブランド整合性**：全レイヤーにおける統一されたブランド価値の維持

---

## 🎯 このリポジトリの目的

1. **戦略の可視化**：事業全体の構造を俯瞰し、各レイヤーの関係性を明確化
2. **開発の効率化**：各事業レイヤーに対応するコード・ツール・スクリプトの統合管理
3. **拡張性の確保**：新規事業・新レイヤーが追加されても破綻しない設計
4. **意思決定の支援**：戦略立案・リスク整理のための壁打ちパートナーとしての機能

## 📝 EA検証スクリプト用ガイドライン

本プロジェクトでは、EA検証コンテンツの制作において以下の思想を基準とします。

### 基本コンセプト
**「データベース（定量情報）＋ストーリー（定性情報）」のハイブリッド構成**

- 視聴者に誤解なく・過信させず情報を届ける
- 専門性と共感性を両立させる
- 言い切りすぎない構成により、自己判断力を育てる

### スクリプト構成テンプレート

#### 1. オープニング（信頼性＋ギャップ提示）
- 紹介者・販売背景を明示
- LPなどとのギャップを提示
- 検証する価値があることを端的に伝える

#### 2. なぜ検証したのか（動機と共感づくり）
- 視聴者の疑問を代弁
- 期待と不安の「ちょうど間」に立つトーン

#### 3. ロジックの解説（EAの性格を翻訳）
- 専門用語を「性格」「役割」「戦い方」に翻訳
- 守り方・攻め方・トレード頻度のイメージを提供

#### 4. バックテストの月別成績
- ブレ幅・波・メンタル負荷に焦点
- 負け月も隠さず提示

#### 5. リスクパート（含み損・最大連敗）
- 「どの程度耐えられるか？」という問いかけ形式
- 証拠金設計とメンタル耐性の重要性を強調

#### 6. チャートでリスクの高い場面を確認
- "ヒヤリとする瞬間"を可視化
- 数字では伝わりにくいリスクを共有

#### 7. 曜日別傾向の確認
- 稼働曜日調整のための材料提供
- ロジックとの相性解説

#### 8. クロージング＆まとめ
- 気づき・学びの要約
- 自己責任・運用判断の重要性を明示

### トーン＆フレーズのガイドライン

**共感 × 分析 × 中立性のバランス**

✅ 推奨フレーズ：
- 「見た目の成績と、運用中の感覚には差があります」
- 「数字だけでは見えないリスクがありました」
- 「もちろん相場状況やロット設定によって変わる部分もありますが…」

❌ 避けるべき表現：
- 過度な断定表現
- 誇大表現
- EAや販売者への一方的な肯定/否定

**シンプル2フォルダ構成の理由：**
- 各プロジェクト（代理店案件等）は全て壁打ちの派生として扱う
- 全てをこのリポジトリで管理する必要はなく、必要に応じて別管理も可能
- 拡張性を保ちながら、実用性を優先

## 🚀 セットアップ

### 前提条件
- Node.js 18.x 以上
- Python 3.9 以上
- Git

### インストール

```bash
# リポジトリのクローン
git clone https://github.com/your-username/YouTube_HAKASE_EALAB.git
cd YouTube_HAKASE_EALAB

# 依存関係のインストール（Webアプリ開発の場合）
npm install

# Python環境のセットアップ（分析ツール開発の場合）
python -m venv venv
source venv/bin/activate  # Windowsの場合: venv\Scripts\activate
```
